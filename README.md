## ‚öôÔ∏è How It Works

### 1. Search Configuration
Users define:
- Research query (e.g., *plant disease detection*)
- Publication year range
- Number of topics to extract
- Reference descriptions defining relevance

### 2. Paper Collection
The notebook automatically retrieves papers from:
- OpenAlex
- arXiv  

Papers are filtered based on publication year and abstract availability.

### 3. Relevance Scoring
Each abstract is:
- Cleaned and preprocessed
- Converted into semantic embeddings
- Compared against user-provided reference descriptions  

A **relevance score** is assigned, and only papers above a chosen threshold are retained.

### 4. Analysis & Visualization
On the filtered dataset, the system attempts to:
- Perform topic modeling
- Extract key research terms
- Visualize publication trends
- Generate topic-wise paper distributions
- Create topic trend heatmaps over time

All results are saved to the `outputs/` folder.

## ‚ö†Ô∏è Known Issues

- After relevance filtering, remaining abstracts may be:
- Too few in number
- Dominated by common or non-informative words
- This can result in:
- Empty vocabulary errors during topic modeling
- Failed keyword extraction
- Missing or incomplete visualizations

These issues usually occur when the relevance threshold is set too high.
